I/A คือใคร
I/A (Issuer / Acquirer)
BA ที่สมัครกับ Nexum มี 2 บทบาท · สามารถเป็นได้ทั้งสองพร้อมกัน หรือบทบาทใดบทบาทหนึ่งก็ได้
บทบาทหน้าที่หลักเครื่องมือ
Issuer
ปล่อยกู้เครดิตให้ User · จัดการ Contract Template · monitor ลูกหนี้ · ดูผลกำไร P&L
I/A BackOffice
Issuer section
Acquirer
หาร้านค้ามาเข้าร่วมรับชำระด้วยเครดิต · จัดการร้านค้า · ดูรายได้จาก MDR
I/A BackOffice
Acquirer section
Concept หลักที่ควรรู้
IBC
Income-Backed Credit
หักชำระหนี้อัตโนมัติจากเงินที่ถูก tag ว่า income เท่านั้น ผ่าน Smart Contract ไม่แตะเงินโอนทั่วไป
Platform cap รวมทุก Issuer ≤ XX% · Payroll คือ use case แรก
MDR
Merchant Discount Rate
ค่าธรรมเนียมจาก C2M transaction แบ่ง 3 ส่วนอัตโนมัติ: Interchange Fee + Network Fee + Acquirer Fee
ใช้เฉพาะ C2M · อัตราตามสัญญาที่ตกลงกัน
MCB
Mana Credit Bureau
ระบบ credit scoring กลาง Issuer ขอดู credit history ของ User ได้ก่อนปล่อยกู้ User ต้อง consent ก่อน
ระยะเวลา: one-time / 30 วัน / 90 วัน · ทุก access ถูก log

Mana App · User

8
